# Candidate Screening Table Template

Copy this file to `.hj-skill-local/candidate-fit-tracker/candidate-screening-table.<context-id>.local.md` when starting a real client screening table. Do not commit filled local tables.

Rename the role score columns to match the selected role context when useful. Keep one row per candidate per role context.

## Table Rules

- `候选人`: Candidate name. If unknown, use the resume filename or a stable placeholder.
- `岗位Context`: Stable context id from the role context file, such as `client-role-group` or `backend-platform-roles`.
- `简历来源 / 文件名`: Source platform, file name, pasted text label, or user-provided source.
- `结论`: Use `强推`, `可聊`, `备选`, or `不建议`.
- Role score columns: Use `xx/100`; rename columns to actual role names if the context has stable target roles.
- `岗位x（如果有补充）`: Use only for adjacent roles or extra fit dimensions. Leave blank when not needed.
- `核心理由`: Use `1. ...<br>2. ...<br>3. ...`; each reason must cite candidate evidence.
- `主要风险`: Use `1. ...<br>2. ...`; write `暂无明显风险` only when supported.
- `建议动作`: State the next recruiting action.
- `是否建议电话聊`: Use `是`, `否`, or `先补信息`.
- `电话重点确认`: Use `1. ...<br>2. ...<br>3. ...`; focus on evidence gaps or risks.
